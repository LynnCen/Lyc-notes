一、Import Map 基础概念 ​

1.1 什么是 Import Map ​

Import Map（导入映射）是 WICG 提出的一个提案，用于控制 JavaScript 的模块加载行为。

核心价值：

  • ✅ 支持裸模块导入（Bare Module Imports）
  • ✅ 浏览器原生支持
  • ✅ 运行时集成
  • ✅ 现代前端工程的趋势

浏览器支持：

  • Chrome 89+ 稳定版开始默认支持
  • 其他现代浏览器逐步支持

1.2 基本使用 ​

问题：裸模块导入失败 ​

传统方式（会报错）：

javascript
import _ from 'lodash';
_.clone({})
1
2

错误信息：

错误原因：

  • 浏览器不知道 'lodash' 指向哪个文件
  • 需要完整的 URL 或相对路径

解决方案：使用 Import Map ​

添加 Import Map 后即可工作：

html
<script type="importmap">
{
  "imports": {
    "lodash": "https://esm.dancf.com/npm:lodash-es@4.17.21/lodash.js"
  }
}
</script>

<script type="module">
import _ from 'lodash';
_.clone({})
</script>

2.1 缓存问题 ​

传统方案的痛点 ​

期望：

  • ✅ 所有 URL 唯一
  • ✅ 长时间缓存（强缓存）
  • ✅ 最快的请求是无请求

传统做法：

  • 使用文件内容 Hash（如 app.abc123.js）
  • 保证 URL 唯一性

问题：

依赖关系链：
app.js
  ├─ utils.js
  │   └─ lodash.js ← 更新
  └─ components.js

lodash 更新
   ↓
utils.js 内容变化（import 语句中的 hash 变了）
   ↓
app.js 内容变化（import 语句中的 hash 变了）
   ↓
所有文件缓存失效！

2.2 兼容性处理 ​

浏览器支持情况 ​

浏览器支持版本市场占有率
Chrome89+✅ 高
Edge89+✅ 高
Safari16.4+⚠️ 中
Firefox108+⚠️ 中

问题：需要支持低版本浏览器

解决方案：ES Module Shims ​

ES Module Shims 是一个基于 WebAssembly 的高性能垫片（Polyfill）。

特点：

  • ✅ 生产环境可用
  • ✅ 基于 WebAssembly 词法分析
  • ✅ 性能优秀
  • ✅ 完整支持 Import Map 规范

使用示例：

javascript
if (!HTMLScriptElement.supports || !HTMLScriptElement.supports('importmap')) {
  // 不支持 Import Map，加载垫片
  document.head.appendChild(
    Object.assign(document.createElement('script'), {
      src: 'https://esm.dancf.com/npm:es-module-shims@1.6.2/dist/es-module-shims.js',
      crossorigin: 'anonymous',
      async: true,
      onload() {
        // 使用 importShim 加载模块
        importShim('https://cdn.dancf.com/gdesign/view/static/js2/d7ca4e8b.js');
      },
    }),
  );
} else {
  // 原生支持，直接使用
  import('https://cdn.dancf.com/gdesign/view/static/js2/d7ca4e8b.js');
}

三、ESM CDN 介绍 ​

3.1 什么是 ESM CDN ​

esm.dancf.com 是 Growing Web 推出的 CDN 服务。

定位：

  • 📦 以 npm 作为托管中心
  • 🎯 原生 ES Module 格式
  • 🚀 专为 Import Map 设计

3.2 与 JSPM 的对比 ​

相同点：

  • ✅ 功能相同
  • ✅ 都支持 npm 包
  • ✅ 都输出 ESM 格式

优势：

对比项esm.dancf.comga.jspm.io
访问速度（国内）🚀 快⚠️ 慢
稳定性（国内）✅ 高⚠️ 低
时效性✅ 立即可用⚠️ 需要构建队列
私有化部署✅ 支持❌ 不支持

3.3 为什么要自建 CDN ​

核心原因：

需求现有方案问题自建解决方案
国内访问速度jspm 在国外，访问慢国内部署，访问快
可靠性依赖第三方自主可控
实时性需要构建队列即发即用
扩展性无法定制完全可定制
私有化不支持支持企业私有部署

四、WPM 工具 ​

4.1 什么是 WPM ​

WPM（Web Package Manager）是 Growing Web 推出的包管理工具。

定位：

  • 📦 基于 Import Map 标准
  • 🌐 管理远程 CDN 包
  • 🔗 也可以映射到本地包

与 npm 的对比：

特性npmWPM
包来源本地 node_modules远程 CDN
锁文件package-lock.jsonimportmap.json
关系基础互补

核心价值：

  • ✅ 简化 SaaS 部署
  • ✅ 简化私有化部署
  • ✅ 无需构建即可更新依赖

5.2 明确对等依赖 ​

什么是对等依赖 ​

对等依赖（peerDependencies）：提示宿主环境安装指定的依赖。

目的：

  • ✅ 避免版本冲突
  • ✅ 共享相同的依赖
  • ✅ 减少包体积

配置示例：

json
{
  "name": "@gaoding/vue-components",
  "peerDependencies": {
    "vue": "^2.7.0"
  }
}

工作原理：

项目结构：
my-app/
  ├─ vue@2.7.13 (项目安装)
  └─ @gaoding/vue-components
       └─ 声明 peerDependencies: vue ^2.7.0

结果：
@gaoding/vue-components 使用项目的 vue@2.7.13
不会重复安装 vue

七、实践中的问题 ​

7.1 浏览器插件干扰 ​

问题描述 ​

Import Map 要求：

  • <script type="importmap"> 必须在第一个 <script type="module"> 之前

问题场景：

  • 极少数浏览器插件会在 HTML 顶部注入 <script type="module">
  • 导致 Import Map 失效

示例图：

解决方案 ​

检测代码：

javascript
(function () {
  try {
    var current = document.querySelector('script[type=importmap]');
    var selector = 'script[type=module]';
    
    while (
      (current =
        current.previousElementSibling ||
        (current.parentElement && current.parentElement.previousElementSibling))
    ) {
      if (current.matches(selector) || current.querySelector(selector)) {
        // 检测到插件干扰
        window.prompt(
          '您的某个浏览器插件阻止了' +
            document.domain +
            '的运行，请找到并关闭它。\n复制链接并在新的窗口打开设置界面进行关闭。',
          'chrome://extensions/',
        );
        break;
      }
    }
  } catch (error) {
    // 忽略错误
  }
})();
