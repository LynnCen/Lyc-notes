三、Layer高亮实现原理 ​

3.1 高亮需求分析 ​

需求：

  • 当鼠标悬停在某个layer上时，显示该layer的边界
  • 边界要清晰可见，不受背景影响
  • 性能要好，不能卡顿

挑战：

  • Canvas没有原生的"描边"API（stroke只能描路径，不能描图像）
  • 需要处理透明区域（只描非透明部分的边界）
  • 需要适配不同尺寸的mask

3.2 Canvas描边的技术困境详解 ​

在深入实现细节之前，我们需要理解为什么这个看似简单的需求在技术上却很有挑战性。

Canvas的两种绘制模式 ​

Canvas API提供了两种完全不同的绘制方式：

1. 矢量绘制（Vector Drawing）

这是Canvas的传统强项，使用路径（Path）API来绘制几何图形：

  • moveTo(), lineTo() - 绘制线条
  • arc(), arcTo() - 绘制圆弧
  • bezierCurveTo() - 绘制贝塞尔曲线
  • rect() - 绘制矩形

这些方法的特点是：它们定义的是"形状的数学描述"，而不是具体的像素。完成路径定义后，可以使用：

  • stroke() - 描边（绘制轮廓）
  • fill() - 填充（填充内部）

2. 位图绘制（Bitmap Drawing）

这是处理图像数据的方式：

  • drawImage() - 将图像绘制到canvas
  • putImageData() - 直接操作像素数据
  • getImageData() - 读取像素数据

这些方法的特点是：它们处理的是"已经确定的像素"，不是数学形状。

我们的困境：Mask是位图，不是矢量 ​

SAM模型返回的mask数据是像素级别的分割结果，它本质上是一张位图（bitmap）：

  • 每个像素要么透明（不属于该分区）
  • 要么不透明（属于该分区）

这些mask存储在Canvas中，是纯粹的像素数据。它们不是矢量路径，没有"边界线"的数学描述，只有一堆像素点。

为什么不能直接用stroke()？ ​

Canvas的stroke()方法只能描绘路径，而不能描绘位图。我们有的是这样的数据：

像素数据（mask）：
[0, 0, 0, 255, 255, 255, ...]  // 0=透明，255=不透明

2

而stroke()需要的是这样的数据：

路径数据（path）：
moveTo(10, 20)
lineTo(30, 40)
bezierCurveTo(50, 60, 70, 80, 90, 100)
...

理论上的解决方案1：将位图转换为矢量路径

可以通过算法将mask的边界转换为矢量路径（比如使用Marching Squares算法），然后用stroke()描边。

但这个方案的问题是：

  • ❌ 算法复杂度高，性能开销大
  • ❌ 对于复杂形状（如头发丝），会生成大量的控制点
  • ❌ 需要额外的库或手写复杂算法
  • ❌ 实时响应（hover时）会造成卡顿

理论上的解决方案2：逐像素绘制边界

遍历mask的每个像素，检测边界点（透明像素旁边的不透明像素），然后逐个绘制边框像素。

但这个方案的问题是：

  • ❌ 需要双重循环遍历所有像素，性能极差
  • ❌ 绘制出的边框没有抗锯齿效果，很粗糙
  • ❌ 边框宽度难以控制（只能是1像素的整数倍）

我们的巧妙解决方案：内阴影技术 ​

既然正面进攻不可行，我们换一个思路：不直接描边，而是生成一个"看起来像边框"的视觉效果。

关键洞察：内阴影在边缘的效果，视觉上等同于边框

想象这样一个类比：

  • 一个不透明的图形，如果在边缘内侧有一圈阴影
  • 从视觉上看，这圈阴影就像是"边框"

这就是我们使用的技术：不是真的描边，而是在mask边缘生成内阴影。

3.3 内阴影技术的实现原理 ​

核心思路 ​

项目使用**内阴影（Inset Shadow）**技术实现高亮，整体流程是：

1. 将layer.maskCanvas作为"遮罩"（定义哪些区域需要效果）
2. 利用Canvas的shadowBlur API在边缘生成模糊阴影
3. 通过destination-in混合模式，将阴影限制在mask区域内
4. 重复叠加多次，增强边框可见度
5. 将最终结果绘制到selectionCanvas上显示
1
2
3
4
5

为什么这个方案可行？ ​

Canvas提供了一个强大的阴影API：

  • shadowBlur - 阴影的模糊半径
  • shadowColor - 阴影的颜色
  • shadowOffsetX/Y - 阴影的偏移

这个API原本是用来给图形添加投影效果的（类似CSS的box-shadow），但我们可以"滥用"它来创造边框效果。

关键技巧：

  1. 使用mask作为遮罩，限制阴影只出现在mask区域内
  2. 设置shadowOffsetX/Y = 0，让阴影不偏移
  3. 设置较大的shadowBlur，让阴影在边缘扩散
  4. 填充整个区域，触发阴影效果
  5. 由于混合模式限制，阴影只会出现在边缘（中心被遮挡）

  targetCtx.drawImage(insetShadowCanvas, 0, 0, targetCanvas.width, targetCanvas.height);

// utils/canvas.ts (56-109行)
function addInsetShadowToCanvas(
  sourceCanvas: HTMLCanvasElement,
  targetCanvas: HTMLCanvasElement,
  options: { blur: number; color: string; offsetX: number; offsetY: number },
) {
  const { width, height } = targetCanvas;
  const resultCanvas = createCanvas(width, height);
  const resultCtx = resultCanvas.getContext('2d')!;

  // 步骤3.1：绘制源图像（作为遮罩）
  resultCtx.drawImage(sourceCanvas, 0, 0, width, height);

  // 步骤3.2：设置混合模式为"destination-in"
  // 效果：只保留源图像的非透明区域
  resultCtx.globalCompositeOperation = 'destination-in';
  resultCtx.drawImage(sourceCanvas, 0, 0, width, height);

  // 步骤3.3：恢复混合模式，绘制阴影
  resultCtx.globalCompositeOperation = 'source-over';

  // 步骤3.4：应用阴影滤镜（模拟内阴影）
  resultCtx.shadowBlur = options.blur;
  resultCtx.shadowColor = options.color;
  resultCtx.shadowOffsetX = options.offsetX;
  resultCtx.shadowOffsetY = options.offsetY;

  // 步骤3.5：绘制填充（触发阴影）
  resultCtx.fillStyle = options.color;
  resultCtx.fillRect(0, 0, width, height);

  // 步骤3.6：应用颜色（可选，用于预览）
  const colorCanvas = colorizeCanvas(resultCanvas, AUTO_MASK_HOVER_FILL_COLOR);
  resultCtx.clearRect(0, 0, width, height);
  resultCtx.drawImage(colorCanvas, 0, 0, width, height);
  cleanCanvas(colorCanvas);

  return resultCanvas;
}

3.5 关键技术点 ​

1. 混合模式：destination-in ​

typescript

1

作用：

  • 只保留目标canvas中与源图像重叠的非透明区域
  • 相当于"遮罩"效果
  • 确保阴影只在mask区域内显示

2. 内阴影模拟 ​

typescript
ctx.shadowBlur = options.borderWidth;
ctx.shadowColor = options.borderColor;

原理：

  • Canvas的shadowBlur会在绘制内容的边缘生成模糊阴影
  • 通过填充整个canvas，在mask边缘生成光晕效果
  • 模拟"内阴影"的视觉效果

3. 重复绘制强化 ​

  • 多次叠加阴影，使边框更明显
  • 增强视觉效果

当前实现：

  • ✅ 离屏canvas：使用临时canvas，避免污染主canvas
  • ✅ 及时清理：使用cleanCanvas释放内存

3.8 高亮实现的深度解析（详细文字说明） ​

为什么Canvas描边这么困难？ ​

在Canvas中，如果要给矩形或圆形描边非常简单，但我们面对的mask是像素位图——成千上万个像素点组成的不规则形状（人物轮廓、头发丝、手指等），根本没有简单的几何路径可以描述。

核心矛盾： Canvas的stroke()只能描路径，不能描位图的边界。

虽然理论上可以通过算法将位图边界提取为矢量路径，但这会带来：计算量大、路径复杂、实时性差、抗锯齿差等问题。在用户hover时需要立即响应，没有时间做复杂计算。

内阴影技术的巧妙思路 ​

既然正面描边很难，我们换个思路：能不能创造一个"看起来像边框"的视觉效果？

如果在mask的边缘内侧生成一圈模糊的颜色，从视觉上看，这就是"边框"！这就是内阴影（Inset Shadow）技术的核心思想。

算法的逐步工作原理 ​

步骤1：创建沙盒环境（临时canvas）

就像做菜要先准备案板，我们创建临时画布（resultCanvas），在上面完成所有操作，避免"污染"目标画布。这样万一出错可以安全回退。

步骤2：建立"不可逾越的边界"（destination-in混合模式）

typescript
resultCtx.globalCompositeOperation = 'destination-in';
resultCtx.drawImage(sourceCanvas, 0, 0, width, height);
1
2

这一步看起来奇怪——为什么把同一个图像再画一遍？

想象你在纸上画了人物轮廓，要给轮廓涂色，但颜色绝对不能超出轮廓线。你会先用透明胶带把轮廓线外的区域全部封住。

destination-in混合模式就是这个"透明胶带"，它的规则是：只保留与mask重叠的不透明区域。虽然视觉上没变化，但canvas已经被"加密"了——后续任何绘制都会被限制在mask区域内，建立了一个"不可逾越的边界"。

步骤3：配置阴影参数

typescript
resultCtx.shadowBlur = 4; // 模糊半径4像素
resultCtx.shadowColor = '#33C8E6'; // 青色
resultCtx.shadowOffset = 0; // 不偏移
1
2
3

这一步是在"设置魔法咒语"。我们告诉Canvas："接下来如果有东西被画出来，请自动给它加上阴影"。但这只是设置，还没有真正画东西。

关键点：

  • shadowBlur控制模糊程度，值越大扩散越远
  • shadowColor用青色（不是传统的黑色投影），因为要做高亮
  • shadowOffset = 0不偏移，让阴影围绕对象均匀扩散

当阴影不偏移时，它会在对象边缘均匀扩散，这在视觉上就是"边框"！

步骤4：触发阴影效果

typescript
resultCtx.fillRect(0, 0, width, height);
1

用青色填充整个canvas。等等，这不会把整个画布都涂成青色吗？

不会！因为步骤2的"透明胶带"保护，只有mask的不透明区域会被涂色。

阴影的触发机制： Canvas的阴影只有在"绘制操作"发生时才会触发。fillRect就是这个触发动作。Canvas会在被填充区域的边缘自动生成阴影。

为什么阴影只在边缘？ 阴影向内外两侧扩散，外侧被mask边界截断，内侧保留下来形成"内阴影"，而中心被填充颜色覆盖看不到阴影。

步骤5：强化效果（重复绘制50次）

typescript
for (let i = 0; i < 50; i++) {
  insetShadowCtx.drawImage(insetShadowCanvas, 0, 0, width, height);
}
1
2
3

这看起来很奇怪——把canvas画到自己身上50次？

想象用水彩笔画画：一次涂抹颜色很淡，涂50次颜色非常鲜艳。

问题： 初始阴影太淡（因为模糊算法会降低不透明度） 解决： 通过叠加绘制累积不透明度，让边框从淡青色变成清晰的青色边框

50次是经验值，在"清晰度"和"性能"之间取得平衡。

边框宽度的自适应计算 ​

typescript
borderWidth: Math.max(1, (2 * layer.maskCanvas.width) / hoverCanvasRef.value!.width) *
  pixelRatio.value;
1
2

这个公式的目的：让边框在任何缩放级别和任何设备上都"看起来一样粗"。

如果固定4像素：原始mask 2000px宽时边框占0.2%，缩小到500px时边框还是4px，占比变成0.8%，看起来变粗了4倍！

解决方案：

  • 第一部分：根据缩放比例调整（mask缩小4倍，边框就放大8倍）
  • 第二部分：Math.max(1, ...)确保至少1像素
  • 第三部分：* pixelRatio适配高DPI屏幕（Retina屏幕翻倍）

最终结果：无论什么设备、什么缩放，边框的视觉粗细都保持一致。

完整流程的视觉变化 ​

步骤视觉效果核心变化
步骤1白色人物轮廓复制mask到临时canvas
步骤2白色人物轮廓（看似没变）内部建立边界规则（关键！）
步骤3-4白色人物轮廓配置阴影参数
步骤5青色轮廓 + 淡青色边缘光晕触发阴影，生成内边框（关键！）
步骤6青色轮廓 + 清晰的青色边框强化50次，边框明显（关键！）

关键转折点在步骤2（建立边界）、步骤5（触发阴影）、步骤6（强化效果）。

这个方案的优势 ​

  1. 性能好： 利用Canvas原生阴影API，GPU加速，实时响应
  2. 效果好： 自然的模糊边缘，抗锯齿，视觉舒适
  3. 适应性强： 可处理任意复杂形状（头发丝、手指等）
  4. 实现简单： 不需要复杂的边界提取算法
  5. 可控性强： 边框宽度、颜色、模糊度都可灵活调整

核心洞察： 通过"滥用"Canvas的阴影API，将传统的投影效果转化为边框效果。这种"跨界使用"正是算法巧妙之处。
