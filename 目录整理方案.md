### 目标
统一使用英文 **kebab-case** (小写+连字符)

## 四、特别注意

### ⚠️ AI 目录需要特殊处理

因为 macOS 文件系统**不区分大小写**，`AI/` 和 `ai/` 被视为同一个，所以需要**两步重命名**：

```bash
# ❌ 错误方式（会导致内容丢失）
mv AI ai

# ✅ 正确方式（两步重命名）
mv AI ai_temp    # 第1步：改为完全不同的名字
mv ai_temp ai    # 第2步：改为目标名字
```

### ✅ 确保不丢失文件

- computerBasics 的 `index.md` 和 `data.ts` 要保留
- AI 的所有子目录（Dify、mcp、posts、机器学习、深度学习）都要保留
- project 下的 gaoding、tmm 等项目文档全部保留

## 五、执行步骤（简化）

### Step 1: 创建新目录结构
```bash
cd docs
mkdir -p fundamentals frontend devops ai blog/articles blog/work-experience 
mkdir -p certifications/software-exam certifications/university-courses 
mkdir -p resources about
```

### Step 2: 移动文件（分批执行，每步验证）

**批次1: Fundamentals**
```bash
mv 组成原理 fundamentals/computer-organization
mv 计网 fundamentals/computer-networks
mv os fundamentals/operating-systems
mv dataStructure fundamentals/data-structures
mv algorithm fundamentals/algorithms
mv 408 fundamentals/exam-408
mv computerBasics/index.md fundamentals/
mv computerBasics/data.ts fundamentals/
rmdir computerBasics
```

**批次2: Frontend**
```bash
mv Javascript frontend/javascript
mv typescript frontend/
mv browser frontend/
mv react frontend/
mv Vue frontend/vue
mv Next frontend/next
mv webpack frontend/
mv Git frontend/git
mv node frontend/
mv architecture frontend/
mv designPatterns frontend/design-patterns
mv utils frontend/
```

**批次3: AI (特殊处理)**
```bash
# 两步重命名
mv AI ai_temp
mv ai_temp ai

# 移动 sam
mv sam ai/
```

**批次4: 其他**
```bash
# DevOps
mv Shell devops/shell

# Blog
find posts -type f -exec mv {} blog/articles/ \;
mv project/* blog/work-experience/

# Certifications
mv softExam certifications/software-exam
mv fdu certifications/university-courses/
mv english certifications/university-courses/

# Resources & About
mv nav/* resources/
mv mine/* about/
```

### Step 3: 更新路径引用
```bash
# 更新 fundamentals/data.ts
sed -i '' 's|/dataStructure/|/fundamentals/data-structures/|g' fundamentals/data.ts
sed -i '' 's|/algorithm/|/fundamentals/algorithms/|g' fundamentals/data.ts
# ... 其他路径更新
```

### Step 4: 提交
```bash
git add -A
git commit -m "refactor: reorganize directory structure"
```
