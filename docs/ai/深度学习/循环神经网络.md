# 循环神经网络（RNN）

循环神经网络（Recurrent Neural Network，RNN）是专门处理**序列数据**（文本、语音、时序信号）的神经网络。与全连接网络不同，RNN 通过**隐藏状态**在时间步之间传递信息，从而捕捉序列中的时间依赖关系。

**文档结构：** RNN 结构与计算 → 时序反向传播（BPTT）→ 梯度消失问题 → LSTM → GRU → Seq2Seq → 知识小结。

---

## 一、RNN 结构与计算

### 网络结构

```text
     h_{t-1}
        ↓
x_t → [RNN Cell] → h_t → y_t
        ↑
     同一组权重 W（参数共享）
```

与普通全连接网络的关键区别：**当前时刻的隐藏状态 $h_t$ 不仅取决于当前输入 $x_t$，还取决于上一时刻的隐藏状态 $h_{t-1}$。**

### 数学公式

**隐藏状态更新：**

$$h_t = f(W_h h_{t-1} + W_x x_t + b_h)$$

**输出：**

$$y_t = g(W_y h_t + b_y)$$

其中：

| 符号 | 含义 |
| ---- | ---- |
| $x_t$ | $t$ 时刻的输入（如一个词的词向量） |
| $h_t$ | $t$ 时刻的隐藏状态（记忆） |
| $y_t$ | $t$ 时刻的输出 |
| $W_h, W_x, W_y$ | 可学习权重（所有时间步共享） |
| $f$ | 隐藏层激活函数（常用 tanh 或 ReLU） |
| $g$ | 输出层激活函数（Softmax 等） |

**参数共享：** 所有时间步使用同一组 $W_h, W_x$，使 RNN 可以处理任意长度的序列。

### RNN 按展开方式分类

| 类型 | 结构 | 典型应用 |
| ---- | ---- | -------- |
| 一对多（one-to-many） | 单输入 → 序列输出 | 图像描述生成 |
| 多对一（many-to-one） | 序列输入 → 单输出 | 情感分类 |
| 多对多（many-to-many） | 序列输入 → 序列输出 | 机器翻译（Seq2Seq） |

## 二、时序反向传播（BPTT）

BPTT（Backpropagation Through Time）是标准反向传播在**时间轴上的展开版本**。

### 三个步骤

1. **前向计算**：按时间顺序 $t=1,2,\ldots,T$ 计算每个时刻的 $h_t$ 和 $y_t$，并计算损失 $\mathcal{L} = \sum_t \mathcal{L}_t$。
2. **反向计算误差**：从 $t=T$ 向 $t=1$ 方向，用链式法则计算每个时刻对损失的梯度 $\partial \mathcal{L}/\partial h_t$。
3. **计算权重梯度**：累加所有时间步对同一权重的梯度（因为参数共享），再用 SGD/Adam 更新。

**梯度沿时间链式传播：**

$$\frac{\partial \mathcal{L}}{\partial h_t} = \frac{\partial \mathcal{L}}{\partial h_T} \cdot \prod_{k=t}^{T-1} \frac{\partial h_{k+1}}{\partial h_k} = \frac{\partial \mathcal{L}}{\partial h_T} \cdot \prod_{k=t}^{T-1} W_h \cdot \text{diag}(f'(h_k))$$

## 三、RNN 的问题：长程依赖

### 梯度消失

当序列很长（如 $T=100$）时，反向传播需要乘以 $T$ 个 $W_h$ 矩阵。若 $\lvert W_h\rvert < 1$，乘积指数衰减 → **前几个时间步的梯度趋近于 0，远处信息被遗忘**。

### 梯度爆炸

若 $\lvert W_h\rvert > 1$，乘积指数增大 → 参数更新不稳定（可用梯度裁剪解决）。

**根本原因**：RNN 的隐藏状态是**覆盖式更新**——$h_t$ 直接被新内容覆盖，无法长期保留重要信息。LSTM 通过"门控"机制解决这一问题。

---

## 四、长短期记忆网络 LSTM

LSTM（Long Short-Term Memory）由 Hochreiter & Schmidhuber 于 1997 年提出，通过引入**细胞状态（Cell State）$c_t$** 和三个门控机制，解决长程依赖问题。

### 核心思想

$c_t$ 是贯穿时间步的"高速公路"，门控单元精确控制哪些信息被保留、被遗忘、被输出。

```text
c_{t-1} ──────────────────────────────→ c_t（细胞状态，长期记忆）
              ↑ forget  ↑ input           ↓ output
h_{t-1} → [遗忘门] [输入门] [输出门] → h_t（隐藏状态，短期记忆）
x_t ──────────────────────────────────↗
```

### 三个门 + 一个候选状态

**1. 遗忘门（Forget Gate）**：决定从细胞状态中丢弃多少旧信息（输出 0=完全遗忘，1=完全保留）。

$$f_t = \sigma(W_f [h_{t-1}, x_t] + b_f)$$

**2. 输入门（Input Gate）**：决定将多少新信息写入细胞状态。

$$i_t = \sigma(W_i [h_{t-1}, x_t] + b_i)$$

**候选细胞状态**（新信息内容）：

$$\tilde{c}_t = \tanh(W_c [h_{t-1}, x_t] + b_c)$$

**3. 细胞状态更新**：遗忘一部分旧信息 + 写入一部分新信息。

$$c_t = f_t \odot c_{t-1} + i_t \odot \tilde{c}_t$$

**4. 输出门（Output Gate）**：决定输出多少细胞状态作为隐藏状态。

$$o_t = \sigma(W_o [h_{t-1}, x_t] + b_o)$$

$$h_t = o_t \odot \tanh(c_t)$$

其中 $\odot$ 表示逐元素乘法（Hadamard 积），$\sigma$ 为 Sigmoid 函数。

### LSTM 为什么解决梯度消失

细胞状态 $c_t$ 的更新中包含**加法项** $f_t \odot c_{t-1}$（而非乘法覆盖），梯度可直接沿 $c_t$ 链回传，不会被指数衰减。这类似于 ResNet 的残差连接。

## 五、门控循环单元 GRU

GRU（Gated Recurrent Unit，2014 年）是 LSTM 的简化版，将遗忘门和输入门合并为**更新门**，去掉独立的细胞状态，参数更少，训练更快。

### 计算公式

**重置门**（控制上一隐藏状态的使用比例）：

$$r_t = \sigma(W_r [h_{t-1}, x_t] + b_r)$$

**更新门**（控制新旧状态的融合比例）：

$$z_t = \sigma(W_z [h_{t-1}, x_t] + b_z)$$

**候选隐藏状态：**

$$\tilde{h}_t = \tanh(W_h [r_t \odot h_{t-1}, x_t] + b_h)$$

**最终隐藏状态：**

$$h_t = (1 - z_t) \odot h_{t-1} + z_t \odot \tilde{h}_t$$

### LSTM vs GRU

| 特性 | LSTM | GRU |
| ---- | ---- | --- |
| 门数量 | 3 个（遗忘、输入、输出） | 2 个（重置、更新） |
| 状态 | $c_t$（细胞状态）+ $h_t$ | 仅 $h_t$ |
| 参数量 | 较多 | 较少（约 LSTM 的 75%） |
| 训练速度 | 较慢 | 较快 |
| 效果 | 长序列稍占优 | 短中序列相当 |

## 六、Seq2Seq（序列到序列）

Seq2Seq（Sequence to Sequence）是一种基于 RNN/LSTM 的**编码器-解码器架构**，将一个变长序列映射为另一个变长序列。

### 整体结构

```text
输入序列:  x_1  x_2  x_3  x_4
              ↓
         [编码器 Encoder]（LSTM）
              ↓
         语义向量 c（context vector）—— 压缩整个输入序列的信息
              ↓
         [解码器 Decoder]（LSTM）
              ↓
输出序列:  y_1  y_2  y_3
```

### 编码器（Encoder）

逐步读取输入序列 $x_1, x_2, \ldots, x_T$，最终隐藏状态 $h_T$ 作为整个序列的语义表示（上下文向量 $c = h_T$）。

$$h_t^{enc} = \text{LSTM}(x_t, h_{t-1}^{enc})$$

### 解码器（Decoder）

以 $c$ 作为初始隐藏状态，自回归地生成输出序列（每次以上一步的输出作为当前输入）：

$$h_t^{dec} = \text{LSTM}(y_{t-1}, h_{t-1}^{dec})$$

$$P(y_t | y_{<t}, c) = \text{Softmax}(W h_t^{dec} + b)$$

### Seq2Seq 的瓶颈

将整个输入序列压缩为**固定长度**的向量 $c$，当序列很长时信息丢失严重。**注意力机制（Attention）** 解决了这一问题：解码每个词时动态关注编码器不同位置的输出，而不仅依赖最后一个隐藏状态。

### 典型应用

| 应用 | 输入 | 输出 |
| ---- | ---- | ---- |
| 机器翻译 | 源语言句子 | 目标语言句子 |
| 文本摘要 | 长文本 | 摘要 |
| 语音识别 | 音频序列 | 文字序列 |
| 对话系统 | 问题 | 回答 |

## 七、知识小结

| 知识点 | 核心内容 | 易混淆点 | 难度 |
| ------ | -------- | -------- | ---- |
| RNN 结构 | $h_t = f(W_h h_{t-1} + W_x x_t + b)$，参数跨时间步共享 | 与全连接区别：有"记忆"$h_t$ | ⭐⭐ |
| BPTT | 反向传播在时间轴上展开，梯度沿时间累加 | 序列越长，乘积链越长 | ⭐⭐⭐ |
| 梯度消失 | $\lvert W_h\rvert<1$ 时多步连乘趋零，远处信息丢失 | 根本原因：覆盖式更新 | ⭐⭐⭐ |
| LSTM | 细胞状态 $c_t$ + 遗忘/输入/输出三门，加法更新保留梯度 | $c_t$ vs $h_t$ 的区别 | ⭐⭐⭐ |
| GRU | 简化 LSTM，重置门 + 更新门，参数更少 | GRU 无独立细胞状态 | ⭐⭐⭐ |
| Seq2Seq | 编码器压缩输入 → 上下文向量 $c$ → 解码器生成输出 | 瓶颈问题：$c$ 长度固定 | ⭐⭐⭐ |
| Attention | 解码时动态关注编码器各位置（非本文重点，详见 Transformer） | Attention 解决 Seq2Seq 瓶颈 | ⭐⭐⭐⭐ |
