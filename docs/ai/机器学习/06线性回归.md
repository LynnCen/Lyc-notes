### 1. 分析

**模型持久化概念**：当训练数据量达到几百万甚至几千万时，每次预测都重新训练模型是不现实的，因此需要将训练好的模型参数保存下来，这个过程称为**模型持久化**。

**实现方法**：通过 `joblib` 库的 `dump` 和 `load` 方法实现，保存为二进制文件格式。

### 2. 模型保存与加载

|------|------|
|依赖库|需先导入 `joblib` 库|
|保存语法|`joblib.dump(estimator, './model/mylrmodel01.bin')`，第一个参数是训练好的模型对象，第二个参数是保存路径|
|路径规范|建议保存在专门的 `model` 文件夹下，文件名以 `.bin` 结尾|

#### 加载步骤

|项目|说明|
|------|------|
|加载语法|`myestimator2 = joblib.load('./model/mylrmodel01.bin')`|
|使用方式|加载后的模型对象可直接调用 `predict` 方法进行预测|
|完整流程|模型训练 → 评估 → 保存 → 加载 → 预测（标准机器学习项目构建流程）|

### 归一化

#### 一、为什么做归一化和标准化

- **量纲问题**：当特征的单位或大小相差较大时，数值较大的特征会主导模型训练结果。
- **方差影响**：某特征的方差相比其他特征大出几个数量级时，会导致模型无法有效学习其他特征。
- **实例**：男子健康数据中，身高(1.7m)、体重(67kg)、视力(1.5) 单位不同，体重数值明显大于其他特征，容易导致模型偏差。

#### 例题：男子健康状况分析

- 特征：身高(米)、体重(kg)、视力(0.2–2.0)，单位差异明显。
- 影响：体重范围(60–90) 远大于视力(0.6–1.8)，模型会误认为数值大的特征更重要。
- 解决：通过归一化/标准化消除量纲影响，使各特征具有可比性。

#### 二、归一化原理

**核心公式**：将数据映射到 $[0, 1]$ 区间

$$
x_{norm} = \frac{x - x_{min}}{x_{max} - x_{min}}
$$

**扩展到任意区间 $[m_i, m_x]$：**

$$
x_{scaled} = \frac{x - x_{min}}{x_{max} - x_{min}} \cdot (m_x - m_i) + m_i
$$

**计算步骤：**

1. 计算每列最小值和最大值
2. 用上式得到 $[0, 1]$ 区间的值
3. 若需其他区间，再用扩展公式做线性变换

**示例**：数值 90，$min=60$，$max=90$ 时，$(90-60)/(90-60)=1$；数值 60 则转换为 0。

#### 三、数据归一化 API

```python
from sklearn.preprocessing import MinMaxScaler

# 关键参数：feature_range，指定缩放范围，默认为 (0, 1)
transformer = MinMaxScaler()
data = transformer.fit_transform(X)  # 同时计算并应用变换
```

**使用流程：**

1. 实例化：`transformer = MinMaxScaler()`
2. 拟合并转换：`data = transformer.fit_transform(X)`

#### 四、应用案例

#### 例题：特征预处理并预测

- 数据：二维数组，每行一个样本，每列一个特征
- 实例化：默认 $[0, 1]$ 区间
- 转换：调用 `fit_transform`
- 验证：原始数据 `[[90,2,10,40],[60,4,15,45],[75,3,13,46]]` 转换后符合预期

#### 开发技巧

- 用 IDE 自动补全快速导入类，Ctrl+点击查看源码了解参数
- 输入必须是二维数组；默认不修改 `feature_range` 时即为 $[0,1]$
- `fit_transform` 会同时计算统计量并应用转换

#### 五、知识小结（归一化）

| 知识点 | 核心内容 | 考试重点/易混淆点 | 难度 |

| 特征预处理的目的 | 解决特征单位和量纲差异导致的偏差，使不同特征可比 | 量纲问题、方差差异导致偏向大数值特征 | ⭐⭐ |

### 标准化

#### 一、数据标准化

**核心概念**：将原始数据转换为**均值为 0、标准差为 1** 的标准正态分布。

**数学形式：**

$$
x_{std} = \frac{x - \mu}{\sigma}
$$

其中：

- $\mu$（缪 /mu/）：均值，决定分布中心
- $\sigma$（西格玛 /sigma/）：标准差，反映离散程度
- $\sigma^2$：方差，标准差的平方

**工程要点：**

- 面试时希腊字母发音要准确（μ: 缪，σ: 西格玛）
- 标准化后约：68% 在 $[\mu-\sigma, \mu+\sigma]$，95% 在 $[\mu-2\sigma, \mu+2\sigma]$，99.7% 在 $[\mu-3\sigma, \mu+3\sigma]$

**代码示例：**

```python
from sklearn.preprocessing import StandardScaler

transformer = StandardScaler()
data = transformer.fit_transform(data)

# 关键属性
transformer.mean_  # 各列均值
transformer.var_   # 各列方差（注意是方差，不是标准差）
```

**与归一化对比：**

- 归一化（MinMaxScaler）：受异常值影响大
- 标准化（StandardScaler）：对异常值更鲁棒

#### 二、规律总结

| 对比项 | 归一化 | 标准化 |
| ------ | ------ | ------ |
| 依据 | 最大值、最小值 | 均值、标准差 |
| 公式 | $(x - min)/(max - min)$ | $(x - \mu)/\sigma$ |
| 异常值 | 敏感 | 鲁棒性强 |
| 适用 | 小数据、分布简单 | 大数据、存在异常值 |
| 结果范围 | 默认 $[0, 1]$ | 无固定范围，呈正态 |

**记忆口诀：**

- 「归一看极差，标准看分布」
- 「小数据用归一，大数据用标准」

#### 例题：填空归纳

- 第一空：归一化公式中的 **最大值、最小值** 易受异常值影响，鲁棒性差
- 第二空：标准化在 **样本数量大** 时对异常值不敏感

易错点：混淆方差 `var_` 和标准差；误以为标准化后数据一定在 $[0,1]$（实际无固定范围）。

#### 三、知识小结（标准化）

| 知识点 | 核心内容 | 考试重点/易混淆点 | 难度 |
| ------ | -------- | ----------------- | ---- |
| 标准化方法 | 均值为 0、方差为 1，公式 $(x - \mu)/\sigma$ | μ、σ 的读音；方差与标准差关系 | ⭐⭐ |
| 归一化方法 | 缩放到固定范围（默认 0–1），依赖 min/max；API：`MinMaxScaler` | 异常值影响、鲁棒性 | ⭐⭐ |
| 正态分布特性 | 3σ 法则（68%/95%/99.7%）；μ 定位置，σ 定离散；标准正态 μ=0, σ=1 | 区间与比例对应关系 | ⭐⭐⭐ |
| API 对比 | 标准化：`StandardScaler()`；归一化：`MinMaxScaler()` | 适用场景：标准化→大数据/异常值 | ⭐⭐ |
| 工程术语 | 鲁棒性≈泛化性；方差=var；标准差=std | 面试时希腊字符发音要准确 | ⭐ |

## 线性回归简介与概念

### 1. 核心要素与学习重点

- **核心要素**：线性回归的分类与定义、API 使用、损失函数概念、评估方法。
- **学习重点**：损失函数在深度学习和面试中常被问到，需掌握模型训练使用的损失函数形式。

### 2. 例题：预测体重问题

- **思路**：用已知身高–体重数据建立回归方程，预测新身高对应的体重。
- **示例数据**：5 组 (160cm/56.3kg, 166cm/60.6kg, 172cm/64.9kg, 178cm/69.2kg, 180cm/75kg)。
- **预测目标**：176cm 对应体重（示例预测约 65.4kg）。
- **数学本质**：寻找最佳拟合直线 $\hat{y} = kx + b$。
- **特点**：预测结果为近似值，非精确值。

### 3. 线性回归定义

**定义**：用回归方程对**特征值（自变量）**与**目标值（因变量）**的关系建模。

**数学表达：**

- 基础形式（一元）：$\hat{y} = wx + b$，其中 $w$ 为权重（斜率），$b$ 为偏置（截距）。
- 多元形式：$\hat{y} = w_1 x_1 + w_2 x_2 + \cdots + w_d x_d + b$。
- 矩阵形式：$\hat{\boldsymbol{y}} = \boldsymbol{X}\boldsymbol{w} + b$，涉及 $\boldsymbol{X}$ 的转置与维度约定（常把 $b$ 并入 $\boldsymbol{w}$，$\boldsymbol{X}$ 增加一列 1）。

**应用示例**：用房屋面积、位置、楼层、朝向等特征预测房价。

### 4. 线性回归分类

| 类型 | 说明 | 示例 |
| ------ | ------ | ------ |
| 一元线性回归 | 单特征 | 仅用身高预测体重 |
| 多元线性回归 | 多特征 | 用面积、位置等多特征预测房价 |

实际工作中多元回归更常见。

### 5. 常见应用场景

- 钢轨伸缩长度与温度；昆虫鸣叫次数与天气；GDP 与双十一销售额等。
- 共同点：特征与目标在散点图上呈大致**线性趋势**。

### 6. 概念小结

- **核心关系**：特征值与目标值之间的**线性关系**，本质是 $\hat{y} = \boldsymbol{w}^{\top}\boldsymbol{x} + b$ 的变体。
- **分类**：一元 vs 多元。
- **评估**：回归用 MSE/MAE/RMSE 等，与分类（如 ACC）、聚类（如 CH）不同。

## 线性回归 API 与损失函数

### 1. 线性回归 API 介绍

**核心步骤与代码：**

```python
from sklearn.linear_model import LinearRegression

# 准备数据：特征 X 为二维（每行一样本），目标 y 为一维
X = [[160], [166], [172], [178], [180]]
y = [56.3, 60.6, 64.9, 69.2, 75.0]

estimator = LinearRegression()
estimator.fit(X, y)
pred = estimator.predict([[176]])

# 关键属性
estimator.coef_       # 斜率 w（一元时为标量）
estimator.intercept_  # 截距 b
```

**易错点**：特征必须是二维结构，如 `[[160],[166],...]`，不能是一维 `[160,166,...]`。

### 2. 损失函数

**基本概念：**

- **误差**：$e_i = \hat{y}_i - y_i$（预测值减真实值）。
- **损失函数**：又称代价函数/目标函数，用于衡量预测效果；优化目标是找到使损失函数最小的参数。

**最小二乘法（误差平方和）：**

$$
J(w,b) = \sum_{i=1}^{m} (\hat{y}_i - y_i)^2 = \sum_{i=1}^{m} (w x_i + b - y_i)^2
$$

固定 $b$ 后可展开为关于 $w$ 的二次函数，求导令 $\frac{\partial J}{\partial w}=0$ 得极值点；当二次项系数为正时，该点为最小值。

**三种常用形式：**

| 名称 | 公式 | 特点 |
| ------ | ------ | ------ |
| MSE（均方误差） | $\displaystyle \frac{1}{m}\sum_{i=1}^{m}(y_i - \hat{y}_i)^2$ | 放大大误差，对异常值敏感 |
| MAE（平均绝对误差） | $\displaystyle \frac{1}{m}\sum_{i=1}^{m}\lvert y_i - \hat{y}_i\rvert$ | 鲁棒，不受异常值主导 |
| RMSE（均方根误差） | $\displaystyle \sqrt{\frac{1}{m}\sum_{i=1}^{m}(y_i - \hat{y}_i)^2} = \sqrt{\text{MSE}}$ | 量纲与 $y$ 一致 |

记忆：MSE = 误差平方和的**均值**；MAE = 误差**绝对值**的均值；RMSE = MSE 开根号。

### 3. 线性回归求解需要什么

- **数据**：特征 $\boldsymbol{x}$ 与目标 $y$，满足大致线性关系；可为单特征或多特征。
- **模型**：假设函数 $\hat{y} = w x + b$（一元）或多元线性回归。
- **损失**：常用 MSE / MAE / RMSE；优化时多采用 MSE（最小二乘）。
- **优化方法**：**梯度下降**（迭代逼近，适合大规模）或**正规方程**（求偏导得解析解，适合小规模）。

### 4. 知识小结（API 与损失函数）

| 知识点 | 核心内容 | 考试重点/易混淆点 | 难度 |
| ------ | -------- | ----------------- | ---- |
| 线性回归基础 | API 流程：导入→准备数据→实例化→fit→predict | 特征必须二维；coef_/intercept_ | ⭐⭐ |
| 损失函数概念 | 误差=预测−真实；损失函数衡量偏差（代价/目标函数） | 最小二乘与 MSE 的关系 | ⭐⭐⭐ |
| MSE/MAE/RMSE | MSE=平方和均值；MAE=绝对值和均值；RMSE=√MSE | 公式与量纲、异常值敏感性 | ⭐⭐ |
| 损失函数优化 | 二次函数求导得极值；梯度下降/正规方程 | 截距固定时的推导 | ⭐⭐⭐⭐ |
| 求解方法 | 梯度下降：迭代；正规方程：解析解 | 适用场景与稳定性 | ⭐⭐⭐ |

## 正规方程

### 1. 一元线性回归

**损失函数**：衡量真实值与预测值差异，为优化指明方向。

$$
J(k, b) = \sum_{i=1}^{m} (\hat{y}_i - y_i)^2 = \sum_{i=1}^{m} (k x_i + b - y_i)^2
$$

给定 5 组身高体重数据，将 $(x_i, y_i)$ 代入上式展开；对 $k$、$b$ 分别求偏导并令其为 0，联立解得最优 $k$、$b$。

### 2. 多元线性回归

**模型**：$\hat{y} = \boldsymbol{w}^{\top}\boldsymbol{x} + b$，$\boldsymbol{w}$ 为权重向量，$\boldsymbol{x}$ 为 $d$ 维特征。  
**数据集**：$\boldsymbol{X} \in \mathbb{R}^{m \times d}$，第 $i$ 行即第 $i$ 个样本；第 $i$ 个样本预测值 $\hat{y}_i = \boldsymbol{x}_i^{\top}\boldsymbol{w} + b$（常将 $b$ 并入 $\boldsymbol{w}$，$\boldsymbol{X}$ 增加一列 1）。  
**损失函数（最小二乘）**：

$$
J(\boldsymbol{w}) = \|\boldsymbol{X}\boldsymbol{w} - \boldsymbol{y}\|^2 = (\boldsymbol{X}\boldsymbol{w} - \boldsymbol{y})^{\top}(\boldsymbol{X}\boldsymbol{w} - \boldsymbol{y})
$$

**正规方程解**：对 $\boldsymbol{w}$ 求导令其为 0，得（在 $\boldsymbol{X}^{\top}\boldsymbol{X}$ 可逆时）：

$$
\boldsymbol{w}^* = (\boldsymbol{X}^{\top}\boldsymbol{X})^{-1} \boldsymbol{X}^{\top} \boldsymbol{y}
$$

**房价预测应用要点**：特征矩阵 $\boldsymbol{X}$ 常含面积、卧室数、房龄等，第一列可设为 1（对应截距）；先算 $\boldsymbol{X}^{\top}\boldsymbol{X}$，求逆后与 $\boldsymbol{X}^{\top}\boldsymbol{y}$ 相乘得 $\boldsymbol{w}$。

### 3. 总结

- **核心公式**：$\boldsymbol{w} = (\boldsymbol{X}^{\top}\boldsymbol{X})^{-1}\boldsymbol{X}^{\top}\boldsymbol{y}$，一步得到最优解。
- **特点**：无需迭代、不需学习率；适合特征维度不高；高维时求逆代价大。
- **与梯度下降**：正规方程直接解析解；梯度下降多步迭代逼近。

### 4. 知识小结（正规方程）

| 知识点 | 核心内容 | 考试重点/易混淆点 | 难度 |
| ------ | -------- | ----------------- | ---- |
| 正规方程 vs 梯度下降 | 正规方程一步求解析解；梯度下降迭代逼近 | 效率与适用规模 | ⭐⭐⭐ |
| 最小二乘与偏导 | 损失为误差平方和，对 k、b 求偏导联立 | 推导过程 | ⭐⭐ |
| 正规方程推导 | $\boldsymbol{w}=(\boldsymbol{X}^{\top}\boldsymbol{X})^{-1}\boldsymbol{X}^{\top}\boldsymbol{y}$ | $\boldsymbol{X}^{\top}\boldsymbol{X}$ 需可逆 | ⭐⭐⭐⭐ |
| 多元与矩阵 | 权重向量、特征矩阵、转置与维度 | 维度匹配 | ⭐⭐⭐ |
| 正规方程局限性 | 需矩阵求逆，特征多时计算贵、易受噪声影响 | 高维与病态问题 | ⭐⭐⭐⭐ |

## 梯度下降

### 1. 梯度与梯度下降

**梯度定义**：函数在某点**变化率最快**的方向。

- **单变量**：梯度即导数（切线斜率），表示函数增长最快的方向。  
- **多变量**：梯度为各变量偏导数组成的向量 $\nabla J = (\frac{\partial J}{\partial \theta_1}, \ldots, \frac{\partial J}{\partial \theta_d})^{\top}$。

**参数更新公式：**

$$
\theta_{new} = \theta_{old} - \alpha \cdot \frac{\partial J}{\partial \theta}
$$

- $\alpha$：学习率（步长），常取 0.001～0.01；过大易震荡，过小收敛慢。  
- 负号：梯度是上升方向，要最小化故沿负梯度方向更新。

### 2. 单变量梯度下降示例

**例题**：求 $J(\theta) = \theta^2$ 的极小值。

- 导数：$\frac{dJ}{d\theta} = 2\theta$。  
- 设初始 $\theta_0 = 1$，学习率 $\alpha = 0.4$。  
- 更新：$\theta_{t+1} = \theta_t - 0.4 \times 2\theta_t = 0.2\,\theta_t$。  
- 迭代：1 → 0.2 → 0.04 → 0.008 → …，趋近 0。  
- 收敛判断：$|\theta_{new} - \theta_{old}|$ 或 $|\nabla J|$ 足够小则停止。

### 3. 多变量梯度下降示例

**例题**：$J(\theta_1, \theta_2) = \theta_1^2 + \theta_2^2$。

- 偏导：$\frac{\partial J}{\partial \theta_1} = 2\theta_1$，$\frac{\partial J}{\partial \theta_2} = 2\theta_2$。  
- 梯度向量：$\nabla J = (2\theta_1,\, 2\theta_2)^{\top}$。  
- 取起点 $(1, 3)$，$\alpha = 0.1$：  
  - 第一步：$(1,3) - 0.1(2,6) = (0.8, 2.4)$；  
  - 第二步：$(0.8, 2.4) - 0.1(1.6, 4.8) = (0.64, 1.92)$；  
  - 持续迭代趋近 $(0, 0)$。

### 4. 知识小结（梯度下降基础）

| 知识点 | 核心内容 | 考试重点/易混淆点 | 难度 |
| ------ | -------- | ----------------- | ---- |
| 梯度概念 | 单变量=导数；多变量=偏导向量；沿梯度方向函数上升最快 | 负梯度方向用于最小化 | ⭐⭐ |
| 单变量示例 | $J=\theta^2$，$dJ/d\theta=2\theta$，$\theta \leftarrow \theta - \alpha \cdot 2\theta$ | 初始值、学习率 | ⭐⭐ |
| 多变量示例 | $J=\theta_1^2+\theta_2^2$，偏导后同步更新 $\theta_1,\theta_2$ | 同时更新所有参数 | ⭐⭐⭐ |
| 通用原理 | 迭代式 $\theta \leftarrow \theta - \alpha \nabla J$，学习率控制步长 | 学习率过大/过小 | ⭐⭐ |
| 符号 | 损失可用 $J(\theta)$ 或 $L(\theta)$，无本质区别 | 不混淆即可 | ⭐ |

### 5. 梯度下降基本步骤与负号、学习率

**四步流程**：① 初始化参数与学习率 $\alpha$；② 计算当前损失对参数的梯度 $\nabla J$；③ 更新 $\theta \leftarrow \theta - \alpha \nabla J$；④ 若 $|\theta_{new}-\theta_{old}|<\varepsilon$ 或达到最大迭代次数则停止。

**负号意义**：梯度方向是函数**增加**最快的方向；要最小化损失，故沿**负梯度**方向更新（“下山”选最陡下坡）。考题常问：“为什么用负梯度？”——因为我们要的是下降方向。

**学习率**：

- 过小：收敛慢、易陷局部最优。  
- 过大：可能越过最优点（Overshooting）、震荡（Oscillation），甚至发散。  
- 常用范围约 0.001～0.01。

### 6. 应用案例：银行信贷（梯度计算）

**设定**：3 个特征（月薪、存款、房产面积），目标为授信额度；8 条样本。  
**假设函数**：$\hat{y} = w_0 + w_1 x_1 + w_2 x_2 + w_3 x_3$（$x_0=1$ 表截距）。  
**损失函数（带 1/2 便于求导）**：

$$
J(\boldsymbol{w}) = \frac{1}{2m}\sum_{i=1}^{m}(\hat{y}_i - y_i)^2
$$

求导后平方项产生因子 2，与分母 2 抵消，梯度形式更简洁。  
**向量形式**：梯度 $\nabla J = \frac{1}{m}\boldsymbol{X}^{\top}(\boldsymbol{X}\boldsymbol{w} - \boldsymbol{y})$；标量形式对第 $j$ 个权重：$\frac{\partial J}{\partial w_j} = \frac{1}{m}\sum_{i=1}^{m}(\hat{y}_i - y_i)x_{ij}$。  
**更新**：$w_j \leftarrow w_j - \alpha \cdot \frac{\partial J}{\partial w_j}$。批量法每轮用全部样本算平均梯度再更新；迭代多轮直至收敛。

### 7. 梯度下降法分类

| 类型 | 梯度计算方式 | 公式（示意） | 特点 |
| ------ | ------------- | ------------ | ------ |
| 全梯度下降 FGD | 使用**全部**样本梯度求平均 | $\theta \leftarrow \theta - \alpha \frac{1}{m}\sum_{i=1}^{m}\nabla J_i$ | 稳定、计算量大 |
| 随机梯度下降 SGD | 每轮**随机选 1** 个样本的梯度 | $\theta \leftarrow \theta - \alpha \nabla J_i$ | 快、波动大 |
| 小批量 Mini-batch | 每轮用**一小批**样本（如 32） | $\theta \leftarrow \theta - \alpha \frac{1}{\lvert B\rvert}\sum_{i \in B}\nabla J_i$ | 折中，常用 |
| 随机平均梯度 SAG | 用当前样本梯度与**历史梯度均值** | 维护梯度表，用均值更新 | 后期收敛好，需存历史 |

- **FGD**：Full，全样本；适合小数据。  
- **SGD**：Stochastic，单样本；应用广但不稳定。  
- **Mini-batch**：batch_size=1 即 SGD，=m 即 FGD。  
- **SAG**：唯一用到“历史梯度”的；初期可能慢（初始梯度为 0）。

### 8. 知识小结（梯度下降优化与分类）

| 知识点 | 核心内容 | 考试重点/易混淆点 | 难度 |
| ------ | -------- | ----------------- | ---- |
| 梯度下降原理 | 沿负梯度迭代 $\theta \leftarrow \theta - \alpha \nabla J$ | 为什么取负梯度 | ⭐⭐⭐⭐ |
| 损失函数 1/2 | $J=\frac{1}{2m}\sum(\hat{y}-y)^2$，求导后系数简洁 | 加 1/2 的目的 | ⭐⭐⭐ |
| 偏导与向量化 | 标量 $w_j$ 更新用 $\sum_i (\hat{y}_i-y_i)x_{ij}$；向量用 $\boldsymbol{X}^{\top}(\boldsymbol{X}\boldsymbol{w}-\boldsymbol{y})$ | $x_0=1$ 表截距 | ⭐⭐⭐⭐ |
| 收敛判定 | 参数变化量很小或达到最大迭代次数 | 学习率对收敛的影响 | ⭐⭐⭐ |
| FGD vs SGD | 全样本平均 vs 单样本随机 | 稳定性与速度权衡 | ⭐⭐⭐⭐ |

### 9. 梯度下降与正规方程对比

| 对比项 | 正规方程 | 梯度下降 |
| ------ | -------- | -------- |
| 公式 | $\boldsymbol{w}=(\boldsymbol{X}^{\top}\boldsymbol{X})^{-1}\boldsymbol{X}^{\top}\boldsymbol{y}$ | $\theta \leftarrow \theta - \alpha \nabla J$ |
| 学习率 | 不需要 | 需要 $\alpha$ |
| 计算方式 | 一步求解析解 | 迭代逼近 |
| 适用 | 小数据、特征数不多 | 大数据、高维特征（如 28×28 图像） |
| 缺点 | 需 $\boldsymbol{X}^{\top}\boldsymbol{X}$ 可逆、求逆贵、对噪声敏感 | 学习率需调、可能震荡 |

**小结**：梯度 = 单变量导数/多变量偏导向量；各类 GD（FGD/SGD/Mini-batch/SAG）只是“用哪批样本算梯度”不同，更新形式统一为 $\theta \leftarrow \theta - \alpha \cdot$（梯度）。

## 回归模型评估方法

**目的**：用指标衡量预测与真实的差距，判断拟合效果。公式与损失函数相同，但**用途是评估/选模型**，而非训练时的优化目标。

### 1. 三种指标公式

| 指标 | 全称 | 公式 | 特性 |
| ------ | ------ | ------ | ------ |
| MSE | Mean Squared Error | $\displaystyle \frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2$ | 值越小越准；对异常值敏感（误差>1 被放大） |
| MAE | Mean Absolute Error | $\displaystyle \frac{1}{n}\sum_{i=1}^{n}\lvert y_i - \hat{y}_i\rvert$ | 稳定，对异常值不敏感 |
| RMSE | Root Mean Squared Error | $\displaystyle \sqrt{\frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2} = \sqrt{\text{MSE}}$ | 量纲与 $y$ 一致；同样敏感于异常值 |

### 2. API 与示例代码

```python
from sklearn.metrics import mean_absolute_error, mean_squared_error

# 假设 y_true, y_pred 已定义
mae = mean_absolute_error(y_true, y_pred)
mse = mean_squared_error(y_true, y_pred)
rmse = mse ** 0.5
```

### 3. 应用案例要点

- **噪声拟合**：在真实关系（如 $y=2x+5$）上加正态噪声 $\mathcal{N}(0,1)$，通常 RMSE > MAE（平方放大效应）。
- **异常值敏感**：若加入少数大误差点，MAE 增幅较小，RMSE 增幅更大（如示例中 MAE +60%，RMSE +111%）。
- **实践**：MAE 看整体误差水平，RMSE 帮助发现是否存在明显异常点；两者结合使用。

### 4. 知识小结（回归评估）

| 知识点 | 核心内容 | 考试重点/易混淆点 | 难度 |
| ------ | -------- | ----------------- | ---- |
| 评估 vs 损失 | MAE/MSE/RMSE 公式与损失相同，用途为评估/选模型 | 训练用损失优化，评估用指标比较 | ⭐⭐ |
| MAE | 绝对误差均值，稳定、不敏感异常值 | 越小越好 | ⭐⭐ |
| MSE | 误差平方和均值，对异常值敏感 | >1 放大、<1 缩小 | ⭐⭐⭐ |
| RMSE | $\sqrt{\text{MSE}}$，量纲一致 | 比 MAE 更敏感异常值 | ⭐⭐⭐ |
| 指标对比 | MAE 稳、RMSE 显大误差；需结合使用 | 过小可能过拟合 | ⭐⭐⭐⭐ |
| API | `mean_absolute_error`, `mean_squared_error` | 函数名与公式对应 | ⭐⭐ |

## 线性回归 API 与案例小结

### 1. 两种求解方式与 API

| 方法 | 公式 | API | 特点 |
| ------ | ------ | ------ | ------ |
| 正规方程 | $\boldsymbol{w}=(\boldsymbol{X}^{\top}\boldsymbol{X})^{-1}\boldsymbol{X}^{\top}\boldsymbol{y}$ | `LinearRegression(fit_intercept=True)` | 一步得解，适合小规模 |
| 梯度下降 | $\theta \leftarrow \theta - \alpha \frac{\partial J}{\partial \theta}$ | `SGDRegressor(loss='squared_loss', fit_intercept=True, learning_rate='constant', eta0=0.01)` | 迭代优化，适合大规模 |

**参数说明：**

- **fit_intercept**：是否拟合截距（偏置项）。
- **loss='squared_loss'**：损失为误差平方和（最小二乘），与 MSE 差一个常数因子（是否除以 $n$）。
- **learning_rate='constant'**：固定学习率；**eta0**：初始学习率（如 0.01）。另有 `invscaling` 等自适应策略（与迭代次数 $t$ 有关）。
- **属性**：`.coef_` 为回归系数 $\boldsymbol{w}$，`.intercept_` 为截距 $b$。

### 2. 波士顿房价案例要点

- **数据**：506 条样本，13 个特征（犯罪率、住宅用地比例、一氧化氮浓度等），目标 MEDV（房价中位数）。
- **流程**：特征标准化（如 `StandardScaler`）→ 划分训练/测试集 → 选模型（正规方程或 SGD）→ 用 MAE/MSE/RMSE 评估，必要时正则化防过拟合。

### 3. 知识小结（API 与案例）

| 知识点 | 核心内容 | 考试重点/易混淆点 | 难度 |
| ------ | -------- | ----------------- | ---- |
| 正规方程 API | `LinearRegression(fit_intercept)`，$\boldsymbol{w}=(\boldsymbol{X}^{\top}\boldsymbol{X})^{-1}\boldsymbol{X}^{\top}\boldsymbol{y}$ | fit_intercept 含义 | ⭐⭐⭐ |
| 梯度下降 API | `SGDRegressor(loss='squared_loss')`，更新式 $\theta - \alpha \partial J/\partial \theta$ | squared_loss = 最小二乘 | ⭐⭐⭐⭐ |
| 学习率参数 | `learning_rate='constant'`，`eta0=0.01` | constant 与 invscaling 区别 | ⭐⭐⭐⭐ |
| 属性 | 两者均有 `.coef_`、`.intercept_` | 内部计算方式不同 | ⭐⭐⭐ |

## 欠拟合与过拟合

### 1. 概念与误差表现

| 类型 | 定义 | 误差表现 |
| ------ | ------ | ------ |
| 欠拟合 | 模型过于简单，训练/测试都差 | 训练误差大、测试误差大 |
| 过拟合 | 模型过于复杂，记住噪声 | 训练误差小、测试误差大 |

**图形记忆**：横轴模型复杂度，纵轴错误率；左侧欠拟合（双高），中间最优点，右侧过拟合（训练低、测试高）。

### 2. 原因与解决方案

**欠拟合原因**：模型/特征过于简单、学习能力不足。  
**解决**：增加特征（如 $x^2,\, x^3$）、用更复杂模型、特征组合；注意过度会变成过拟合。

**过拟合原因**：特征冗余、数据噪声（bad case）、模型过复杂、数据量不足。  
**解决**：① 清洗数据（分析 bad case）；② 增加数据量；③ 正则化（L1/L2）；④ 降维（如 PCA）。  
**工程顺序**：清洗 → 增数据 → 正则化 → 降维。

### 3. 代码示例：欠拟合 / 正常拟合 / 过拟合

**数据**：$x \sim \text{Uniform}(-3,3)$，$y = 2x^2 + 3x + 1 + \varepsilon$，$\varepsilon \sim \mathcal{N}(0,1)$。

```python
import numpy as np
from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error
from sklearn.preprocessing import PolynomialFeatures

np.random.seed(42)
X = np.random.uniform(-3, 3, size=100).reshape(-1, 1)
y = 2 * X.ravel()**2 + 3 * X.ravel() + 1 + np.random.normal(0, 1, 100)

# 欠拟合：仅线性
lr = LinearRegression().fit(X, y)
print('欠拟合 MSE:', mean_squared_error(y, lr.predict(X)))  # 约 2.4

# 正常拟合：加二次项，模型 y = w0 + w1*x + w2*x^2
X2 = np.hstack([X, X**2])
lr2 = LinearRegression().fit(X2, y)
print('二次拟合 MSE:', mean_squared_error(y, lr2.predict(X2)))  # 约 0.96

# 过拟合：高次项 X, X^2, ..., X^9
X9 = np.column_stack([X**i for i in range(1, 10)])
lr9 = LinearRegression().fit(X9, y)
# 训练 MSE 更小，但测试集或新数据上会变差
```

**要点**：MSE 不是越小越好，过小可能过拟合；要兼顾泛化（看测试误差）。

### 4. 正则化（L1 / L2）

**目的**：在损失函数中加惩罚项，约束权重大小，减轻异常点影响，相当于对特征做筛选或平滑。

**L1 正则化（Lasso）**：

$$
J(\boldsymbol{w}) = \frac{1}{2m}\sum_{i=1}^{m}(\hat{y}_i - y_i)^2 + \alpha \sum_{j=1}^{d}|w_j|
$$

- 对 $|w_j|$ 求导：$w_j>0$ 时导数为 $+1$，$w_j<0$ 时为 $-1$，$w_j=0$ 处不可导（次梯度）。
- **效果**：权重被压向 0，部分权重精确为 0，实现**特征筛选**（稀疏解）。

**L2 正则化（Ridge）**：

$$
J(\boldsymbol{w}) = \frac{1}{2m}\sum_{i=1}^{m}(\hat{y}_i - y_i)^2 + \alpha \sum_{j=1}^{d} w_j^2
$$

- 对 $w_j$ 求导多出一项 $2\alpha w_j$，权重整体缩小但一般不为 0。
- **效果**：权值平滑、减小过拟合，不产生稀疏。

**对比**：L1 稀疏、可做特征选择；L2 平滑、数值稳定。工程上过拟合排查：先 bad case → 数据量 → 正则化 → 降维。

### 5. 知识小结（欠拟合、过拟合与正则化）

| 知识点 | 核心内容 | 考试重点/易混淆点 | 难度 |
| ------ | -------- | ----------------- | ---- |
| 欠拟合 | 模型太简单，训练/测试都差；增加特征或模型复杂度 | 特征不足示例 | ⭐⭐ |
| 过拟合 | 模型太复杂，训练好测试差；清洗、增数据、正则化、降维 | 四步排查顺序 | ⭐⭐⭐ |
