**项目背景**

**技术栈：**
`Vue 2.7` `TypeScript` `React` `Python FastAPI` `Qwen-Agent` `LangGraph` `豆包大模型` `PixiJS` `SSE` `Dify` `Redis` `MCP协议` `编辑器基座化` `Ant Design X`

**核心职责**

1. 🤖 **后端Agent架构设计**（参与架构设计）
   - 参与设计7层架构体系（接入层→服务层→路由层→Agent层→执行层→工具层→基础设施层），采用Plan-and-Execute模式实现复杂任务规划
   - 基于Qwen-Agent框架集成LangGraph状态图编排，设计Router层基于豆包大模型（Doubao-Seed-1.8）实现意图识别和Agent选择，支持AdvancedAgent（复杂任务）和OrdinaryAgent（简单任务）双模式切换
   - 实现动态工具加载机制：工具配置热更新，5秒缓存+动态type()创建工具类，无需重启服务即可上线新工具，支持MCP协议工具动态注册
   - 设计工具级中断恢复方案：InterruptManager + Redis Checkpoint，解决用户稿豆不足等场景的精准恢复，避免重复执行已成功工具
   - 实现工具并行执行优化：基于ThreadPoolExecutor实现最多2个工具并行调用，通过contextvars.copy_context()解决上下文传递，性能提升50%
   - 设计Token成本优化方案：VariableMemoryManager实现URL压缩（135字符→9字符），节省90% Token，大模型调用成本降低20-30%

2. 💬 **前端AI对话系统架构**
   - 设计SSE流式通信架构：TransformStream + XStream流式JSON解析，解决分段JSON问题，实现AI生成内容的实时逐字输出，响应延迟<500ms

   - 设计批量生成追踪机制：consecutiveFunctionCalls追踪算法，统计"生成4张图"等批量任务的完成进度，通过Map数据结构优化时间复杂度至O(n)

   - 实现10+种AI工具的前端调用（抠图/改图/消除/扩图/滤镜/转矢量等），支持复杂任务拆解和多工具链式调用

- 在Tao日本APP电商平台的逆向物流流程中，用户申请退款和退货的场景复杂多样。为了提高用户体验和运营效率，我们对全流程包裹维度的逆向入口进行了优化，确保在特定情况下能够快速、准确地处理用户的退款请求。
- 参与Tao日本站逆向前端基础架构的设计与实施，参加从技术选型、性能优化到组件标准化的全流程开发。

**技术栈：**
`React` `ICE` `TypeScript` `Vite` `Lerna` `SCSS` `Node.js` `跨端H5`

**核心职责**
1. 🛠 **基础架构**
   - 基于 Def 搭建高效研发工作流，降低团队学习与迁移成本，提升开发体验。
   - 设计并实现了一套基于[React的图标自动化生成器](/project/generate_icon)，通过Node.js脚本实现从API数据拉取到React组件生成的端到端自动化，显著提升项目中图标资源的管理效率和工程化水平。

2. 🚀 **性能优化**
   - 设计CSR架构性能优化方案，结合 ZCache 预加载与 [Dataprefetch](/posts/dataPreFetch) 数据预取，优化首屏加载。
   - 设计基于[SSG的骨架屏方案](/posts/skelon)，通过构建时预渲染生成带骨架屏的HTML，结合客户端动态加载真实DOM，有效解决了首屏白屏问题。
   - 通过[ssgShimPlugin插件](/posts/skelon#node环境下如何访问客户端环境变量-ssgshimplugin)在Node环境中模拟客户端环境变量，确保SSG模式下代码的兼容性与稳定性。

3. 📦 **组件库建设**
   - 主导 odin-jp 系列基础物料库建设，收敛原集团 7+ 分散组件库（如 comet-ui、cosmos-base）。
   - 统一封装为轻量化、高内聚的[monorepo](/project/lerna_monorepo)组件库（UI、Hooks、API等），降低维护成本并提升复用率。
   - 实现UI主题色动态配置与 [UnoCSS 预设](/posts/uno-css-jp) 集成，支持响应式布局与移动端优先策略。

4. 📊 **监控与优化**
   - 通过 [AEM埋点系统](/project/aem) 实现业务全链路监控，埋点规范覆盖点击、曝光、页面 SPM 等场景。
   - 设计自动化异常采集方案，结合 initMtop 接口拦截与 AES 加密，保障数据安全性与问题追溯效率。
   - 对接 ADC/PHA 容器能力，实现半浮层、头部隐藏、下拉刷新等原生交互。
   - 通过 Windvane 桥接 Native 模块，提升端内 H5 体验接近原生应用。

**项目成果**
- 通过构建高效研发工作流、自动化图标管理系统和优化的 CSR、SSG 性能，显著提升了用户体验和开发效率。
- 整合的基础物料库和增强的原生交互能力，使应用在不同平台上的运行更加流畅和一致。
- 通过全链路监控和数据加密，确保了数据的安全性和系统的稳定性。
- 开发效率提升30%，系统性能和用户体验获得显著改善。

### [羚羊智能客服工作台](/project/linyang) | 阿里巴巴

**项目描述**

客服工作台是阿里巴巴旗下的一款智能客服产品，是阿里在电商和新零售等领域多年沉淀的一套全渠道智能客服解决方案。提供服务前关键信息识别、智能调度，服务中快速回复、客服辅助、快速标记，服务后数据分析、智能质检的功能。

**技术栈：**`React+Typescript+Agora RTC+Websocket+Mobx+Ant Design+Severos`

**责任描述**：

1. 🎥 **[视频客服](/project/linyang#视频通信系统架构设计)**
   - 基于Agora RTC SDK搭建高并发视频通信系统，结合WebSocket实现信令同步，设计多窗口渲染架构，支持主屏、侧边视频列表、最小化悬浮窗等多形态交互。
   - 客户端采用分层状态机设计(Context+useReducer)，封装useCallControl Hook管理通话操作（接通/挂断/静音/设备切换），用户端通过Mobx响应式状态流实现双端实时状态管理。
   - 实现WebRTC级联重连策略（断网5秒内自动重连）、通话超时熔断（30秒无响应自动挂断），系统异常率下降15%。开发设备健康度检测模块，预检摄像头/麦克风权限、网络质量评分（基于RTT与丢包率），增强异常拦截准确率。

2. 🤖 **[AI智能客服](/project/linyang#ai智能客服)**
   - 设计[多模态消息协议](/project/linyang#_1-消息类型-messageprotocol-ts)，新增chaAIMessage（闲聊）、docAiMessage（文档解析）、summaryAiMessage（会话总结）三类消息，通过[SSE流式渲染](/project/linyang#_2-sse流式渲染系统)引起实现实时内容逐字输出。
   - 开发[轮训调度器（usePolling）](/project/linyang#_3-轮询调度器)，结合指数退避算法动态调整请求间隔，压缩大模型响应时间。
   - 搭建Agent-lab可视化配置系统，支持拖拽式AI技能编排、话术模版动态绑定等。

### TMMTMM实时聊天软件(对标土耳其版微信) | 四川盛通智联网络科技有限公司 


**项目描述：**
独立负责 PC 端全链路开发，从零搭建对标土耳其版微信的跨平台实时聊天应用（支持 Win/Mac），DAU 超 10 万，上线后土耳其用户覆盖率提升 20%，负责TMM后台管理系统、移动端页面及官网的开发和维护，涵盖广告管理、流量监控、每日上线人数统计、包管理、版本管理等功能模块，为负责人及合作伙伴提供数据可视化与管理支持负责维护。

**技术栈：**
`React` `TypeScript` `Electron` `Mobx` `WebSocket` `IndexDB/Dexie` `AWS` `Antd`

**责任描述：**

1. 🔄 **[模块重构与架构设计](/project/tmm/architecture)**
   - 基于[关注点分离原则](/project/tmm/architecture#_1-1-关注点分离)，将聊天模块重构为API（接口聚合）、DB（数据持久化）、Event（跨模块通信）、Data（状态管理）、UI（视图渲染）五层架构，通过依赖注入解耦逻辑。

2. ⚡ **[性能优化](/project/tmm/performance_optimization)**
   - 结合React的memo与[虚拟列表](/project/tmm/performance_optimization#一、虚拟消息列表)实现消息动态渲染，减少不必要的DOM操作。
   - 利用[Mobx的细粒度响应式更新](/project/tmm/performance_optimization#二、mobx状态优化)，优化复杂状态同步场景。

3. ☁️ **云端资源管理**
   - 采用[流式读写策略](/project/tmm/resources_manner#_1-基本概念)，基于node的FS模块与AWS SDK分片上传接口，[实现大文件下载](/project/tmm/resources_manner#_2-实现架构)、[分片上传](/project/tmm/partFileUpload#分片上传)、[断点续传](/project/tmm/partFileUpload#断点续传)和[并行上传](/project/tmm/partFileUpload#并行上传)。
   - 通过[背压控制](/project/tmm/resources_manner#_3-背压控制)平衡内存与I/O吞吐量，有效解决文件下载卡顿问题。
   - 采用[LUR算法](/algorithm/LRU)管理本地资源和缓存高频接口数据，可拓展[Service Worker](/browser/渐进式网页应用PWA#什么是-service-worker)实现离线资源预加载。

4. 🔌 **实时通信系统**
   - 基于[WebSocket](/计网/websocket)的[心跳包](/计网/websocket#心跳机制)与ACK确认机制保证消息可靠性，集成[自动重连](/计网/websocket#自动重连)与[离线队列补偿](/计网/websocket#离线队列)。
   - 应用[CAP定理](/posts/IM多端一致性)权衡一致性，有效解决多端操作冲突，确保会话状态（置顶/静音）一致性。

5. 🔍 **[本地化搜索引擎](/project/tmm/local_search)**
   - 基于IndexDB的多对象存储特性，设计[联合索引](/project/tmm/local_search#二、联合索引的深度解析)实现跨表查询优化。
   - 通过[位运算](/project/tmm/local_search#三、位运算)快速过滤优先级会话，减少全表扫描开销。
   - 使用[Web Worker](/project/tmm/local_search#四、web-worker在本地搜索引擎中的应用)分离计算搜索与主线程渲染，避免卡顿。

6. ✍️ **编辑器拓展**
   - 基于Quill的Delta模型开发插件，通过MutationObserver监听光标事件。
   - 实现@消息上下文感知，支持O(n)时间复杂度匹配成员。
   - 利用异或运算动态标记关联消息，支持@消息跳转。

7. 🛠 **其他核心功能开发**
   - 基于栈结构（LIFO）实现最近表情动态排序。
   - 通过[防抖](/utils/throttle)与差异对比策略降低存储频率和实现草稿持久化方案。
   - 开发权限系统，基于[RBAC（角色控制访问）](/project/tmm/RBAC)模型设计群组权限体系。
   - 通过位掩码实现细粒度权限组合（如发言/管理/公告编辑）。
   - 通过Electron-builder实现平台签名打包，设计增量热更新方案优化用户体验。
   - 基于Electron封装系统通知、文件拖拽等操作系统级功能。
   - 设计实现二维码登陆系统。
