# 论文重点准备要点（2025-11 预测）

下列为基于近年命题规律提炼的重点备考知识清单。建议按“概念→原理→设计要点→实施步骤→指标与评估→案例话术”路径准备，并配合自己的真实项目沉淀可引用的细节与数据。

## 1. 服务治理与容错设计（含服务网格/网关）
- 核心目标与指标：可用性/延迟/稳定性/成本；SLA/SLO/SLI 与错误预算；依赖分级与优先级。
- 典型策略：
  - 限流（令牌桶/漏桶、并发限流、排队与超时）
  - 熔断（开/半开/闭状态、滑动窗口判定、快速失败与自愈）
  - 降级（静态兜底/读缓存/关闭非关键路径）
  - 重试与退避（指数退避+抖动，重试风暴防护，最大重试预算）
  - 幂等性（幂等键、去重表、幂等令牌、Exactly-once 的工程实现）
  - 舱壁隔离/线程池隔离/连接池隔离；超时控制与超时预算计算
  - 热点保护与背压；请求合并与批处理
- 流量治理：路由/灰度/金丝雀/蓝绿；区域就近；权重与熔断联动；故障注入与演练。
- API 网关 vs 服务网格：
  - 网关职责：认证鉴权、协议与路由、聚合与限流、审计
  - 网格职责：数据面/控制面、细粒度策略下沉、mTLS、可观测性、零信任落地
- 设计要点：
  - 超时与重试配比（端到端超时预算分配）
  - 幂等保障与事务边界（支付/扣款/库存等典型场景）
  - 依赖降级矩阵与SLO对齐；回滚/熔断/降级闭环
- 落地步骤：现状评估→策略选型→规则建模→全链路压测→灰度→监控告警→回滚预案。
- 常见坑：全链路重试风暴；状态服务幂等等价难题；限流排队过长；误触发熔断雪崩。
- 产出物：依赖拓扑与时序图、阈值/配额表、演练SOP。

## 2. 可观测性体系（Metrics/Logs/Traces + SLO/SLI）
- 方法论：三支柱（指标/日志/链路）；RED/USE 指标系；SLO/SLI/错误预算管理。
- 指标设计：可用性/延迟分位数/吞吐/饱和度；容量规划与基线；服务级 vs 业务级指标。
- 日志：结构化与脱敏；样本率与留存；审计合规；灰度与回滚线索。
- 链路追踪：Trace Context（W3C/B3）；采样策略；跨语言/跨边界追踪；N+1/高延时定位。
- 工具栈：OpenTelemetry、Prometheus/Alertmanager、Grafana、Loki、Tempo/Jaeger。
- 告警策略：阈值/异常检测/复合告警；抑制/静默；误报与漏报控制。
- 落地步骤：指标选型→埋点与采集→告警拓扑→SLO评审→演练与复盘。
- 常见坑：仅堆工具不落指标；无基线导致误报；端到端缺失导致定位困难。

## 3. CDC 与实时数据集成
- 采集模式：Binlog/变更流、时间戳/版本列、轮询；全量+增量双通道。
- 语义保证：At-least-once/At-most-once/Exactly-once；两阶段提交、幂等 Sink、事务性 Sink。
- 时序与乱序：事件时间/处理时间；水位线与迟到数据；回溯重放与补数。
- 架构组合：Debezium/Maxwell/Canal → Kafka/Pulsar → Flink/Spark → OLAP/ES/ClickHouse/DW。
- Schema 演进：兼容策略（向后/向前/双向）；Schema Registry；数据版本治理。
- 数据一致性：端到端对账/去重；幂等键；Exactly-once 的边界与成本。
- 数据质量监控：空值/重复/越界/校验规则；DQ 指标、告警与隔离；死信队列与补偿。
- 落地步骤：源端评估→链路设计→语义选择→监控与告警→回溯与演练→变更发布流程。
- 常见坑：DDL 变更导致任务中断；跨 Region 延迟与乱序；回溯污染在线指标。

## 4. 数据治理与数据质量
- 核心概念：元数据（技术/业务）、血缘（表/字段/作业级）、主数据（MDS）、数据标准与口径。
- 质量维度：准确性/完整性/一致性/唯一性/及时性/可追溯性；度量方法与阈值设定。
- 组织与流程：治理委员会/Owner/Steward 职责；变更管理与数据评审；问题闭环（发现→定位→整改→验证）。
- 工具与平台：Data Catalog（Atlas/Amundsen）、质量平台（Great Expectations）、血缘（OpenLineage）。
- 权限与合规：分级分域、最小权限、数据脱敏；日志留存与审计。
- 落地步骤：资产编目→标准与口径→DQ 规则→监控告警→问题管理→复盘优化。
- 常见坑：无统一口径导致跨域冲突；血缘不全影响溯源；规则过多带来维护负担。

## 5. 高可用与容灾架构
- 指标与目标：可用性 N 个 9；RTO/RPO；故障域与Blast Radius 控制。
- 架构形态：冷/温/热备；双活/多活；单AZ/多AZ/多 Region；存算分离与状态复制（同步/半同步/异步）。
- 一致性与切换：主从/仲裁/选主；读写分离；DNS/GSLB/Anycast 流量切换；缓存预热与热点迁移。
- 故障演练与混沌工程：注入/演练/复盘；演练SOP与回滚预案；依赖降级联动。
- 成本与权衡：跨域带宽/存储成本；一致性与延迟折中（CAP/PACELC 视角）。
- 落地步骤：业务连续性等级→架构选型→数据复制→切换预案→演练→监控→复盘。
- 常见坑：一致性误解导致数据分叉；演练缺失；切换窗口与缓存污染。

## 6. 零信任安全架构（可与服务网格/DevSecOps协同）
- 原则：Never trust, always verify；持续评估；最小权限与微隔离。
- 组件：身份与访问管理（IdP/SSO/MFA）、设备/环境感知；策略引擎/执行点（PDP/PEP）；mTLS；ABAC/RBAC。
- 标准与框架：NIST 800-207；BeyondCorp 思想；策略编排与审计闭环。
- 与服务网格结合：服务间 mTLS、细粒度策略下沉；东西向流量控制与可观测性联动。
- 与 DevSecOps：SAST/DAST/IAST/SCA、供应链安全（SBOM/签名）、合规门禁与发布阻断。
- 落地步骤：资产盘点→信任边界→策略建模→渐进式部署→度量与审计。
- 常见坑：策略过细导致可用性受损；证书/密钥治理；跨域互信复杂度。

## 通用答题结构模板（适配三段式）
1) 项目与职责：背景/目标/规模（QPS/数据量/团队/约束）+ 你的角色与 KPI。
2) 原理与方法：核心概念/架构与流程/关键算法或策略/指标与权衡（列出2-3个关键决策点与取舍）。
3) 设计与落地：系统设计图与链路、实施步骤、关键配置与阈值、监控与演练、效果（量化：可用性、延迟、成本、故障率）。

## 项目案例要素清单（答题可复用填空）
- 规模：峰值 QPS、平均延迟 P95/P99、数据日增量/吞吐；关键依赖数量与层级。
- 关键指标：SLO/SLI、错误预算、RTO/RPO、DQ 覆盖率、告警噪声率。
- 关键决策：为什么选 A 不选 B（技术/成本/组织）及权衡数据。
- 演练与复盘：演练频率、故障注入、回滚时长、复盘闭环项。

## 自查清单（考前一页纸）
- 是否能用一张图讲清架构与链路？
- 是否有2个可量化的效果数字（前后对比）？
- 是否准备1个“权衡取舍”的关键决策案例？
- 是否准备“降级/熔断/回滚/演练”的闭环证据？
- 是否准备“数据质量/一致性/安全合规”的证明材料？
